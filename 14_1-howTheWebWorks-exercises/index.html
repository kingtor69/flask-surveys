Part One: Solidify Terminology

What is HTTP?
HyperText Transfer Protocol is standard format for communication between client computers and web servers for web sites. The protocol includes how requests are formatted and send and how responses are format and returned, as well as how the body of the information is formatted to make the web site look the way it does on the client browser.
What is a URL?
A Uniform Resourse Locator is a logically written address that is used to direct your browser where in the world to find the information you want.
What is DNS?
Domain Numbering System is a means of translate the logically-written domain in a URL into a numbered address corresponding to the web server.
What is a query string?
A query string is an option section in a URL (following a question mark) which contains further user-entered information for the server, such as search terms.
What are two HTTP verbs and how are they different?
  1. GET is a request from user to server which does not change any information on the server. The GET request directs the server towards requested information and the format in which the user is expecting to a response.
  2. POST is a request from user which is intended to change information on the server (generally adding it, as a social media post). In addition to the information to be added, the POST request includes information about the format of the POSTed information as well as expectations of response from the server.
What is an HTTP request?
A request is from the user (or client) to the server asking for information. It includes information about what response is expected as well as specifications of how the data is expected to be returned.
What is an HTTP response?
The reponse comes from the server back to the user/client and includes the information requested as well as data about the format of that information and identifying information about the reply (date/time, from whom, &c.)
What is an HTTP header?
An HTTP header is a widget of the information in a request or response which consists of a key and value with the key being what kind of information is included and the value being that information. Examples:
Accept Language fr
Accept application/json
Content-Type text/html; charset=ISO-8859-1
What are the processess that happen when you type “http://somesite.com/some/page.html” into a browser?
  1. the browser first looks for “somesite.com/”’s DNS information from (in order):
    1. it's own cache
    2. the next router upstream
    3. the ISP to which that router is connected
    4. DNS servers
  2. the browser now has an IP address from the DNS listing
  3. the browser then sends a GET request to the IP address using the port 80 (for “http://”)
  4. this GET request is asking the server for “some/page.html,” asking for text/html content in the browser's default language
  5. the server processess this request and starts sending information back to the browser
  6. the browser makes more requests for information contained within the body (links to CSS style sheets, JS scripts, images, &c.
  7. as this information is coming in from the server, the browser starts building the page in front of the user's eyes
